Clase 04

Estadística Descriptiva Univariada

Preparación, limpieza, transformación y exportación de bases de datos en R siguiendo un flujo de trabajo reproducible.

Contenidos de la clase

El objetivo de esta guía práctica es conocer las principales formas de realizar análisis estadístico descriptivo en R, aplicando los conocimientos aprendidos durante el curso.

En detalle, aprenderemos a:

Establecer un flujo de trabajo

Organizar un script .R de manera ordenada y reproducible.

Analizar variables categóricas

Calcular frecuencias, porcentajes y tablas de contingencia.

Analizar variables numéricas

Calcular medidas de tendencia central, dispersión y posición.

Interpretar resultados descriptivos

Traducir los resultados estadísticos a una lectura sustantiva.

En esta práctica trabajaremos con los datos procesados obtenidos en la práctica anterior a partir del Estudio Longitudinal Social de Chile ELSOC, realizado por COES.

Recursos de la práctica

En esta práctica usaremos la base procesada proc_elsoc.RData, construida a partir de la ola 6 del Estudio Longitudinal Social de Chile — ELSOC 2022.

Recuerden que siempre es importante trabajar junto con el manual o libro de códigos de la base de datos. El manual de variables de ELSOC 2022 se puede revisar en el siguiente enlace:

Manual de variables ELSOC 2022

Flujo de trabajo reproducible

Por temas de orden y reproducibilidad, en este curso vamos a separar el trabajo con datos en dos momentos:

Preparación

Corresponde a la limpieza y modificación previa de los datos. En esta etapa se seleccionan variables, se filtran casos, se recodifican respuestas y se construyen nuevas variables.

Análisis

Corresponde al cálculo de estadísticos descriptivos y a la interpretación de los resultados en función de preguntas de investigación.

En términos generales, distinguimos entre datos originales y datos preparados o procesados. Los datos originales provienen directamente de la fuente, mientras que los datos procesados son aquellos que ya fueron adaptados para el análisis.

Los procesos de preparación y análisis vinculados tanto a datos como a resultados se presentan en el siguiente esquema:

Tanto la preparación como el análisis quedan registrados en archivos de código separados. En esta guía nos centraremos en el análisis de datos con R.

El documento de código de análisis tiene, por lo menos, las siguientes partes:

  1. Identificación y descripción general: título, autoría, fecha e información breve sobre el contenido.
  2. Librerías: instalación y carga de paquetes.
  3. Datos: carga de datos procesados.
  4. Exploración: revisión inicial de la base.
  5. Análisis: cálculo de estadísticos descriptivos.

Análisis descriptivo de datos

1. Cargar librerías

Este paso ya lo realizamos en sesiones anteriores. Sin embargo, si trabajamos en una nueva sesión de R, debemos volver a cargar las librerías necesarias.

install.packages("pacman") # para instalar
library(pacman)            # para llamar/cargar

En este práctico utilizaremos los siguientes paquetes:

  1. pacman: facilita la instalación y carga de paquetes.
  2. tidyverse: colección de paquetes para manipulación y análisis de datos.
  3. dplyr: permite seleccionar, transformar y resumir variables.
  4. psych: permite calcular estadísticos descriptivos.
  5. sjmisc: permite calcular frecuencias y otros descriptivos.
  6. sjPlot: permite construir tablas cruzadas o de contingencia.
pacman::p_load(tidyverse, # colección de paquetes para manipulación de datos
               dplyr,     # para manipular datos
               psych,     # para analizar datos
               sjmisc,    # para analizar datos
               sjPlot)    # para tablas de contingencia

options(scipen = 999) # desactivar notación científica
rm(list = ls())       # limpiar el entorno de trabajo

2. Importar datos

Usaremos los datos procesados en la práctica anterior, guardados en la carpeta output.

load("output/proc_elsoc.RData")

También podemos cargar una base directamente desde internet usando url():

load(url("https://github.com/cursos-metodos-facso/descriptiva/raw/main/assignment/output/proc_elsoc.Rdata"))
Note

La función load() proviene de R base y permite cargar bases de datos en formato .RData o .Rdata.

También podemos cargar bases desde internet utilizando la función url().

3. Explorar datos

Antes de comenzar el análisis, conviene revisar la base de datos.

View(proc_elsoc)   # ver datos
names(proc_elsoc)  # nombres de columnas
dim(proc_elsoc)    # dimensiones
str(proc_elsoc)    # estructura de los datos

En este caso, nuestra base de datos procesada tiene 496 casos y 9 variables.

4. Análisis descriptivo para variables categóricas

Cuando trabajamos con variables categóricas, sean nominales u ordinales, podemos utilizar tablas de frecuencias.

Las frecuencias permiten ordenar los datos según los valores o categorías alcanzadas por una variable.

4.1 Medidas de frecuencia

a) Frecuencias absolutas y relativas

Para variables nominales podemos usar tablas de frecuencias absolutas y relativas. Esto también nos permite identificar la moda, es decir, la categoría con mayor cantidad de observaciones.

Una forma sencilla de calcular frecuencias absolutas es mediante la función table():

table(proc_elsoc$sexo)

table(proc_elsoc$ingreso_minimo)

table(proc_elsoc$autor_democ)

Lo anterior entrega la frecuencia absoluta de las variables. Por ejemplo, en cuanto a preferencias entre autoritarismo y democracia, podemos observar en qué categoría se concentra la mayor cantidad de casos.

Para conocer la frecuencia relativa o porcentual, usamos prop.table():

prop.table(table(proc_elsoc$autor_democ)) * 100

Otra forma de calcularlo es utilizando lenguaje orientado a objetos:

freq_table1 <- table(proc_elsoc$autor_democ)

prop.table(freq_table1) * 100

Así, podemos interpretar qué porcentaje de los casos se ubica en cada categoría de respuesta.

5. Frecuencias acumuladas

Cuando trabajamos con variables ordinales, también podemos calcular frecuencias acumuladas.

cumsum(prop.table(table(proc_elsoc$tramo_ingreso)) * 100)

Otra forma de calcularlo es guardando los resultados en objetos:

freq_table2 <- table(proc_elsoc$tramo_ingreso)

freq_table3 <- prop.table(freq_table2) * 100

cumsum(freq_table3)

A partir de este estadístico, podemos observar qué porcentaje acumulado de casos se ubica hasta cierto tramo de ingresos.

También podemos unir todas estas frecuencias en una sola tabla:

tbl3 <- table(proc_elsoc$tramo_ingreso)

cbind(
  Freq = tbl3,
  Relativa = prop.table(tbl3) * 100,
  Acumulada = cumsum(tbl3)
)

Otra manera de calcular frecuencias absolutas, relativas y acumuladas en R es mediante la función frq() del paquete sjmisc:

sjmisc::frq(proc_elsoc$tramo_ingreso)

6. Tablas de contingencia

También podemos cruzar dos variables mediante tablas de contingencia o tablas cruzadas.

Estas tablas permiten observar la distribución conjunta de dos variables categóricas. Además de la frecuencia absoluta en cada celda, podemos calcular porcentajes por fila, columna o celda.

sjt.xtab(proc_elsoc$sexo, proc_elsoc$tramo_edad)

Tabla cruzada con porcentaje de celda:

sjt.xtab(proc_elsoc$sexo, proc_elsoc$tramo_edad,
         show.cell.prc = TRUE,
         show.summary = FALSE)

Tabla cruzada con porcentaje por fila:

sjt.xtab(proc_elsoc$sexo, proc_elsoc$tramo_edad,
         show.row.prc = TRUE,
         show.summary = FALSE)

Tabla cruzada con porcentaje por columna:

sjt.xtab(proc_elsoc$sexo, proc_elsoc$tramo_edad,
         show.col.prc = TRUE,
         show.summary = FALSE)
Note

En una tabla cruzada, los porcentajes por fila y por columna responden preguntas distintas.

El porcentaje por fila permite leer cómo se distribuyen las categorías de la variable de columna dentro de cada fila.

El porcentaje por columna permite leer cómo se distribuyen las categorías de la variable de fila dentro de cada columna.

7. Estadísticos descriptivos para variables numéricas

A diferencia de las variables categóricas, a las variables numéricas, intervalares o de razón, les podemos calcular una mayor cantidad de estadísticos descriptivos.

Entre ellos encontramos:

Tendencia central

Media o promedio y mediana.

Dispersión

Desviación estándar, varianza y rango.

Posición

Mínimo, máximo, cuartiles y percentiles.

Tip

Recordemos que:

  • Las medidas de tendencia central expresan el valor alrededor del cual se sitúa la mayor cantidad de los datos.
  • Las medidas de dispersión permiten observar qué tan próximos o alejados están los valores respecto a un punto central.
  • Las medidas de posición indican en qué lugar de la distribución se encuentra un dato o conjunto de datos en relación con el resto.

8. Medidas de tendencia central

Media o promedio

La media corresponde al promedio aritmético de una variable numérica.

mean(proc_elsoc$ingreso, na.rm = TRUE)

Mediana

La mediana corresponde al valor que divide la distribución en dos partes iguales.

median(proc_elsoc$ingreso, na.rm = TRUE)

9. Medidas de dispersión

Varianza

La varianza mide cuánto se alejan los datos respecto a la media.

var(proc_elsoc$ingreso, na.rm = TRUE)

Desviación estándar

La desviación estándar es la raíz cuadrada de la varianza. Es una de las medidas de dispersión más utilizadas porque está expresada en la misma unidad que la variable original.

sd(proc_elsoc$ingreso, na.rm = TRUE)

Rango

El rango corresponde a la diferencia entre el valor máximo y el valor mínimo.

diff(range(proc_elsoc$ingreso, na.rm = TRUE))

10. Medidas de posición

Primer cuartil Q1

El primer cuartil corresponde al valor bajo el cual se encuentra el 25% de los datos.

quantile(proc_elsoc$ingreso, probs = .25, na.rm = TRUE)

Segundo cuartil Q2 o mediana

El segundo cuartil corresponde al valor bajo el cual se encuentra el 50% de los datos.

quantile(proc_elsoc$ingreso, probs = .50, na.rm = TRUE)

Tercer cuartil Q3

El tercer cuartil corresponde al valor bajo el cual se encuentra el 75% de los datos.

quantile(proc_elsoc$ingreso, probs = .75, na.rm = TRUE)

Percentiles

Los percentiles dividen la distribución en cien partes.

quantile(proc_elsoc$ingreso, probs = seq(0, 1, 0.01), na.rm = TRUE)

11. Resumen de estadísticos con summary()

Podemos obtener rápidamente un resumen de los datos con la función summary() de R base.

summary(proc_elsoc$ingreso)

Con esto podemos ver el mínimo, primer cuartil, mediana, media, tercer cuartil y máximo de la variable.

Aunque es informativo, summary() no entrega todos los estadísticos que podríamos necesitar, como la desviación estándar o la varianza.

12. Resumen de estadísticos con psych

Usando la función describe() del paquete psych, podemos obtener una mayor cantidad de estadísticos descriptivos.

psych::describe(proc_elsoc$ingreso,
                quant = c(.25, .75),
                IQR = TRUE)

Esta función entrega medidas como la media, mediana, media recortada, desviación estándar, rango y recorrido intercuartílico.

El recorrido intercuartílico corresponde a:

IQR = Q3 − Q1

Es decir, mide la dispersión del 50% central de los datos.

13. Resumen de estadísticos con summarise()

Otra manera de obtener los estadísticos descriptivos es utilizando summarise() de dplyr.

Aquí debemos especificar explícitamente qué medidas queremos calcular:

proc_elsoc %>% 
  summarise(
    media = mean(ingreso, na.rm = TRUE),
    mediana = median(ingreso, na.rm = TRUE),
    q1 = quantile(ingreso, probs = .25, na.rm = TRUE),
    q3 = quantile(ingreso, probs = .75, na.rm = TRUE),
    rango = max(ingreso, na.rm = TRUE) - min(ingreso, na.rm = TRUE),
    desviacion_estandar = sd(ingreso, na.rm = TRUE),
    varianza = var(ingreso, na.rm = TRUE)
  )

Esta forma es especialmente útil porque permite construir tablas personalizadas con los estadísticos que necesitamos reportar.

14. Visualización de datos con ggplot2

Una vez que conocemos cómo es la distribución de nuestras variables, podemos graficarlas considerando algunos principios básicos para una buena visualización de datos.

Antes de construir los gráficos, es importante comprender qué es ggplot2 y cuál es su lógica.

ggplot2 es un potente paquete para visualizar datos en R. Su lógica principal es que funciona por capas, comenzando desde lo más central hasta llegar a los detalles estéticos.

Datos

La primera capa corresponde a la base de datos que queremos graficar.

Mapping

La segunda capa define qué variables irán en el eje X, eje Y, color, relleno o grupos.

Geometría

La tercera capa define la forma del gráfico: barras, puntos, líneas, histogramas, boxplot, entre otros.

La estructura básica de un gráfico con ggplot2 es:

ggplot(data = base_de_datos,
       mapping = aes(x = variable_x, y = variable_y)) +
  geom_tipo_de_grafico()

¡Comencemos!

15. Diagrama de barras

La forma más común de presentar variables nominales u ordinales es mediante un gráfico de barras.

En este primer ejemplo iremos capa por capa para entender la lógica de ggplot2. En los siguientes ejemplos lo haremos directamente en conjunto.

Primero especificamos los datos y el mapping:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo))

Luego agregamos la geometría de barras:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo)) +
  geom_bar()

Después podemos agregar color de borde y relleno:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo)) +
  geom_bar(color = "black", fill = "darkblue")

También podemos ajustar la escala del eje Y:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo)) +
  geom_bar(color = "black", fill = "darkblue") +
  scale_y_continuous(limits = c(0, 500))

Agregamos título, nombres de ejes y fuente:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo)) +
  geom_bar(color = "black", fill = "darkblue") +
  scale_y_continuous(limits = c(0, 500)) +
  labs(title = "Situación respecto al ingreso mínimo",
       x = "Situación de ingreso",
       y = "Frecuencia",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")

Finalmente, podemos agregar las frecuencias de cada barra:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo)) +
  geom_bar(color = "black", fill = "darkblue") +
  scale_y_continuous(limits = c(0, 500)) +
  labs(title = "Situación respecto al ingreso mínimo",
       x = "Situación de ingreso",
       y = "Frecuencia",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.") +
  geom_text(aes(label = after_stat(count)),
            stat = "count",
            vjust = 1.5,
            colour = "white")

16. Diagrama de barras agrupado

También podemos diferenciar una variable categórica según otra variable. Para ello indicamos la variable de agrupación dentro del mapping mediante el argumento fill.

Por ejemplo, podemos graficar la situación respecto al ingreso mínimo según sexo:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo, fill = sexo)) +
  geom_bar() +
  labs(title = "Situación respecto al ingreso mínimo, según sexo",
       x = "Situación de ingreso",
       y = "Frecuencia",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.") +
  geom_text(aes(label = after_stat(count)),
            stat = "count",
            colour = "white",
            position = position_stack(vjust = 0.5))

En este caso tenemos un gráfico de barras apiladas según sexo. La agrupación aparece como una leyenda que identifica los grupos.

Si queremos que las barras aparezcan una al lado de la otra, debemos especificarlo dentro de la geometría con position = "dodge2":

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo, fill = sexo)) +
  geom_bar(position = "dodge2") +
  labs(title = "Situación respecto al ingreso mínimo, según sexo",
       x = "Situación de ingreso",
       y = "Frecuencia",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.") +
  geom_text(aes(label = after_stat(count)),
            stat = "count",
            colour = "white",
            vjust = 1.5,
            position = position_dodge(width = .9))

También podemos agrupar por tramo etario:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso_minimo, fill = tramo_edad)) +
  geom_bar(position = "dodge2") +
  labs(title = "Situación respecto al ingreso mínimo, según tramo etario",
       x = "Situación de ingreso",
       y = "Frecuencia",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.") +
  geom_text(aes(label = after_stat(count)),
            stat = "count",
            colour = "white",
            vjust = 1.5,
            position = position_dodge(width = .9))

17. Histogramas

Cuando tenemos variables numéricas o variables con muchas categorías de respuesta, podemos usar histogramas para representar su distribución.

Un histograma permite observar dónde se concentran los datos, si la distribución es simétrica o asimétrica y si existen valores extremos.

ggplot(data = proc_elsoc, 
       mapping = aes(x = edad)) +
  geom_histogram()

Podemos mejorar el gráfico agregando colores, títulos, ejes y fuente:

ggplot(data = proc_elsoc, 
       mapping = aes(x = edad)) +
  geom_histogram(color = "black", fill = "#FA8072") +
  labs(title = "Histograma de frecuencias absolutas: edad",
       x = "Edad",
       y = "Frecuencia",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")

También podemos construir un histograma para la variable ingreso:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso)) +
  geom_histogram(color = "black", fill = "darkred") +
  labs(title = "Histograma de frecuencias absolutas: ingreso",
       x = "Ingreso",
       y = "Frecuencia",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")

18. Diagrama de densidad

Los diagramas de densidad son una forma suavizada de observar la distribución de una variable numérica.

Son útiles para identificar la forma general de la distribución y para contrastar visualmente distribuciones empíricas con distribuciones teóricas.

ggplot(data = proc_elsoc, 
       mapping = aes(x = edad)) +
  geom_density(color = "black", fill = "#FA8072", alpha = 0.8) +
  labs(title = "Diagrama de densidad: edad",
       x = "Edad",
       y = "Densidad",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")

También podemos observar la densidad del ingreso:

ggplot(data = proc_elsoc, 
       mapping = aes(x = ingreso)) +
  geom_density(color = "black", fill = "darkred", alpha = 0.8) +
  labs(title = "Diagrama de densidad: ingreso",
       x = "Ingreso",
       y = "Densidad",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")

19. Gráficos de caja y bigote

Como vimos en clases, los gráficos de caja y bigote permiten observar la mediana, el primer cuartil, el tercer cuartil, el rango intercuartílico, los valores mínimos y máximos, y posibles valores atípicos.

Primero graficamos la distribución de la edad:

ggplot(data = proc_elsoc, 
       mapping = aes(y = edad)) +
  geom_boxplot(fill = "darkblue", alpha = 0.8) +
  labs(title = "Gráfico de caja: edad",
       y = "Edad",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")

Luego podemos graficar la distribución del ingreso:

ggplot(data = proc_elsoc, 
       mapping = aes(y = ingreso)) +
  geom_boxplot(fill = "darkred", alpha = 0.8) +
  labs(title = "Gráfico de caja: ingreso",
       y = "Ingreso",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")

También podemos segmentar un gráfico de caja por una variable categórica. Por ejemplo, podemos observar la distribución del ingreso según tramo etario:

ggplot(data = proc_elsoc, 
       mapping = aes(x = tramo_edad, y = ingreso, fill = tramo_edad)) +
  geom_boxplot(alpha = 0.8) +
  labs(title = "Gráfico de caja: ingreso por tramo etario",
       x = "Tramo etario",
       y = "Ingreso",
       caption = "Fuente: Elaboración propia en base a ELSOC 2022.")
